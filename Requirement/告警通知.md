# 告警与通知系统

## 功能概述
当环境数据超出阈值时发送告警通知。

## 用户场景
- 作为 **用户**，我希望 **当温度过高时收到告警**，以便 **及时处理异常情况**
- 作为 **用户**，我希望 **自定义告警阈值**，以便 **根据实际环境设置合理的告警条件**

## 具体需求

### 需求 1：阈值配置
- **描述**: 允许用户自定义各项数据的告警阈值
- **优先级**: P1
- **可配置项**:
  - 温度告警阈值：
    - 过高告警：> 45°C
    - 过低告警：< -10°C
  - 湿度告警阈值：
    - 过高告警：> 90%
    - 过低告警：< 20%
  - CO 浓度告警阈值：
    - 危险浓度：> 50 ppm
    - 警告浓度：> 30 ppm
- **预期结果**:
  - 在设置中提供阈值配置界面
  - 保存后立即生效

### 需求 2：告警通知
- **描述**: 当数据超出阈值时显示告警
- **优先级**: P0
- **通知方式**:
  - **界面弹窗**: QMessageBox 警告框
  - **状态栏提示**: 持续显示告警信息
  - **颜色变化**: 数据显示变为红色
- **预期结果**:
  - 数据超阈值时界面立即提示
  - 恢复后自动消失
  - 记录告警历史

### 需求 3：声音告警
- **描述**: 触发告警时播放提示音
- **优先级**: P2
- **功能要求**:
  - 可单独开启/关闭声音
  - 使用系统默认提示音
  - 音量可调节
- **预期结果**:
  - 开启后超阈值时播放声音

## UI/UX 设计
- 阈值配置在设置页面"告警"标签页
- 告警使用 QMessageBox
- 状态栏显示当前告警状态指示器

## 验收标准
- [ ] 阈值可以正确配置和保存
- [ ] 数据超阈值时界面正确告警
- [ ] 声音告警功能正常
- [ ] 告警历史可查看

## 技术约束
- 阈值配置保存到 JSON 文件 (Config/alert_thresholds.json)
- 声音使用 QSound 或系统 API
- 告警不阻塞正常操作

### 使用方式
1. 点击顶部导航栏的 **"正常"** 按钮
2. 弹出告警阈值设置对话框
3. 配置各项阈值
4. 点击 **"保存"** 应用配置

### 消息类型
| 消息类型 | 说明 |
|---------|------|
| `sensor.data.updated` | 传感器数据更新 |
| `alert.triggered` | 触发告警 |
| `alert.config.updated` | 配置已更新 |
| `alert.config.get` | 获取当前配置 |
