# 数据持久化与历史记录

## 功能概述
添加数据本地存储和历史记录功能，支持数据导出和分析。

## 用户场景
- 作为 **用户**，我希望 **保存历史数据**，以便 **后续分析和问题追溯**
- 作为 **用户**，我希望 **导出数据为 CSV/Excel**，以便 **在其他工具中分析**

## 具体需求

### 需求 1：本地数据存储
- **描述**: 将传感器数据保存到本地数据库
- **优先级**: P0
- **技术实现**:
  - 使用 SQLite 数据库（轻量、无需额外服务）
  - 存储内容：
    - 时间戳
    - 温度
    - 湿度
    - CO 浓度
    - 光照强度
  - 数据保留策略：最近 30 天
- **预期结果**:
  - 应用关闭后数据不丢失
  - 支持按时间范围查询

### 需求 2：数据导出功能
- **描述**: 支持将历史数据导出为 CSV 格式
- **优先级**: P1
- **功能要求**:
  - 选择时间范围导出
  - 导出格式：CSV、JSON
  - 文件命名规则：GasCarApp_YYYYMMDD_HHMMSS.csv
- **预期结果**:
  - 点击按钮弹出导出对话框
  - 选择时间范围后点击确认即可导出

### 需求 3：数据统计视图
- **描述**: 提供历史数据的统计分析视图
- **优先级**: P2
- **显示内容**:
  - 今日最高/最低/平均温度
  - 今日最高/最低/平均湿度
  - 设备在线时长统计
  - 数据完整率（成功接收/总次数）

## UI/UX 设计
- 在设置页面增加"数据管理"标签页
- 提供"导出最近 24 小时数据"快捷按钮
- 统计信息以卡片形式展示

## 验收标准
- [ ] 数据自动保存到本地数据库
- [ ] 可以导出指定时间范围的数据
- [ ] 导出文件格式正确
- [ ] 统计信息计算正确

## 技术约束
- 数据库路径：应用数据目录/GasCarApp/data.db
- CSV 编码：UTF-8
- 导出线程：不阻塞 UI
