# 消息系统使用指南

## 概述

消息系统是 GasCarApp 的核心通信机制，基于 **发布-订阅模式**，用于各个模块之间的解耦通信。

## 架构

```
┌─────────────────────────────────────────────────────────────┐
│                      MessageManager                          │
│                   (单例，统一管理消息)                         │
├─────────────────────────────────────────────────────────────┤
│  ┌───────────┐  ┌───────────┐  ┌───────────┐              │
│  │ Handler1  │  │ Handler2  │  │ Handler3  │              │
│  └───────────┘  └───────────┘  └───────────┘              │
│       ↑               ↑               ↑                   │
│       └───────────────┴───────────────┘                   │
│                     register()                             │
└─────────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          ↓               ↓               ↓
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ Presenter │   │   View   │   │  Engine  │
    └──────────┘   └──────────┘   └──────────┘
```

## 1. MessageManager (消息管理器)

**特点**：
- 单例模式，整个应用只有一个实例
- 负责注册 Handler 和分发消息

```python
from Src.Message.MessageManager import MessageManager

# 获取单例
manager = MessageManager()  # 总是返回同一个实例
```

## 2. Message (消息)

**结构**：
```python
Message(type: str, payload: any)
```

- `type`: 消息类型（字符串），用于区分不同消息
- `payload`: 消息内容（任意类型）

**示例**：
```python
from Src.Message.Message import Message

# 发送连接状态消息
msg = Message("serial.connection.status", True)

# 发送传感器数据消息
msg = Message("sensor.data.updated", {
    "temperature": 25.5,
    "humidity": 60,
    "co": 10
})

# 发送告警消息
msg = Message("alert.triggered", {
    "type": "temperature",
    "level": "warning",
    "message": "温度过高"
})
```

## 3. MessageHandler (消息处理器接口)

**实现步骤**：

### 步骤1: 继承 MessageHandler

```python
from Src.Message.MessageHandler import MessageHandler
from Src.Message.Message import Message, HandleResult

class MyPresenter(BasePresenter, MessageHandler):
    def __init__(self, view, message_manager: MessageManager):
        super().__init__(view)
        self._message_manager = message_manager
        # 注册为消息处理器
        self._message_manager.register(self)
```

### 步骤2: 实现 handle() 方法

```python
def handle(self, message: Message) -> HandleResult:
    """
    处理消息
    
    Args:
        message: 收到的消息
        
    Returns:
        HandleResult 枚举值:
        - CONSUMED: 消息已处理，不再传递给其他 Handler
        - CONTINUE: 继续传递给下一个 Handler
        - SKIP: 跳过当前 Handler，继续下一个
    """
    if message.type == "serial.connection.status":
        # 处理连接状态消息
        connected = message.payload  # True 或 False
        print(f"连接状态: {connected}")
        return HandleResult.CONSUMED  # 消费消息
    
    elif message.type == "sensor.data.updated":
        # 处理传感器数据
        data = message.payload
        temperature = data.get("temperature")
        # ... 处理逻辑
        return HandleResult.CONSUMED
    
    # 不处理的消息，继续传递
    return HandleResult.CONTINUE
```

## 4. 发送消息

```python
from Src.Message.MessageManager import MessageManager
from Src.Message.Message import Message

manager = MessageManager()

# 发送消息（分发给所有已注册的 Handler）
manager.dispatch(Message("serial.connection.status", True))
```

## 5. 完整示例

### 示例：创建新的 Presenter 并使用消息系统

```python
# MyModulePresenter.py
from Src.MVP.base_presenter import BasePresenter
from Src.Message.MessageHandler import MessageHandler
from Src.Message.Message import Message, HandleResult
from Src.Message.MessageManager import MessageManager

class MyModulePresenter(BasePresenter, MessageHandler):
    def __init__(self, view, message_manager: MessageManager):
        super().__init__(view)
        self._message_manager = message_manager
        self._message_manager.register(self)
    
    def handle(self, message: Message) -> HandleResult:
        if message.type == "custom.event":
            # 处理自定义事件
            data = message.payload
            print(f"收到自定义事件: {data}")
            return HandleResult.CONSUMED
        return HandleResult.SKIP
    
    def do_something(self):
        # 发送消息给其他模块
        self._message_manager.dispatch(
            Message("custom.event", {"key": "value"})
        )
```

### 示例：在按钮点击时发送消息

```python
# 在某个 View 中
def on_button_clicked(self):
    # 发送消息
    self._message_manager.dispatch(
        Message("ui.button.clicked", {"button": "submit"})
    )
```

## 7. 最佳实践

### 7.1 消息类型命名规范

```python
# 推荐
"serial.connection.status"  # 模块.事件.状态
"sensor.data.updated"       # 模块.数据.操作
"alert.triggered"          # 模块.操作

# 不推荐
"status"                   # 太泛
"SERIAL_CONNECTED"        # 用小写字母
```

### 7.2 消息命名空间

```
serial.        # 串口相关
sensor.        # 传感器相关
alert.         # 告警相关
ui.            # UI 相关
domain.        # 业务逻辑相关
```

### 7.3 Handler 注册时机

```python
# ✅ 正确：在 __init__ 中注册
def __init__(self, message_manager):
    self._message_manager = message_manager
    self._message_manager.register(self)  # 立即注册

# ✅ 正确：在创建后注册
presenter = MyPresenter(view, manager)
manager.register(presenter)
```

### 7.4 消息消费

```python
def handle(self, message: Message) -> HandleResult:
    if message.type == "important.event":
        # 只处理一次
        return HandleResult.CONSUMED
    
    # 允许其他 Handler 也处理
    return HandleResult.CONTINUE
```

## 8. 调试技巧

### 8.1 查看已注册的 Handler

```python
# MessageManager 添加调试代码
def dispatch(self, message: Message):
    print(f"[MessageManager] 分发消息: {message.type}")
    print(f"[MessageManager] 接收者数量: {len(self._handlers)}")
    for handler in self._handlers:
        print(f"  - {handler.__class__.__name__}")
    # ... 继续分发
```

### 8.2 追踪消息流

```python
def handle(self, message: Message) -> HandleResult:
    print(f"[{self.__class__.__name__}] 收到消息: {message.type}")
    # ... 处理
    print(f"[{self.__class__.__name__}] 处理完成")
    return HandleResult.CONSUMED
```

## 9. 常见问题

### Q: 消息没有被处理？

A: 检查以下内容：
1. Handler 是否已注册到 MessageManager？
2. 消息类型是否正确？
3. Handler 是否返回了 CONSUMED？

### Q: 需要全局访问 MessageManager？

A: 在应用的入口处创建并传递：
```python
# main.py
message_manager = MessageManager()
main_page = MainPage(message_manager)
```

### Q: 循环发送消息导致死循环？

A: 使用标志位防止重复处理：
```python
def handle(self, message: Message) -> HandleResult:
    if message.type == "my.event" and not self._processing:
        self._processing = True
        # 处理
        self._processing = False
    return HandleResult.CONSUMED
```

## 10. 消息系统流程图

```
   发送消息                    分发消息                    处理消息
   ──────                    ──────                    ──────
     │                         │                         │
     ├─ dispatch() ──────────>│                         │
     │                         │─ handle() ─────────────>│
     │                         │                         ├─ 检查类型
     │                         │                         ├─ 处理逻辑
     │                         │                         └─ 返回结果
     │                         │<─ CONSUMED ─────────────┤
     │                         │                         │
     │                         │  停止分发（已消费）
     │                         │                         │
     ▼                         ▼                         ▼
```

## 总结

1. **MessageManager** 是单例，负责消息分发
2. **Handler** 实现 `MessageHandler` 接口
3. **消息** 由 `type` 和 `payload` 组成
4. 使用 `register()` 注册，`dispatch()` 发送
5. `handle()` 返回 `HandleResult` 控制消息流
