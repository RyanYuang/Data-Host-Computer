# 上位机 ↔ STM32 串口通信协议文档

> **项目**：环境监测控制系统（GasCarApp）  
> **版本**：v1.0  
> **日期**：2026-02-14  

---

## 1. 物理层参数

| 参数 | 值 |
|------|----|
| 波特率 | 115200（OpenPortByName 默认）/ 备用 9600 |
| 数据位 | 8 |
| 校验位 | 无 (None) |
| 停止位 | 1 |
| 编码 | ASCII（纯可打印字符 + `\r\n`） |

---

## 2. 帧格式总览

所有帧均以 **`\r\n`** (0x0D 0x0A) 结尾，上位机和下位机各自以行缓冲方式解析。

| 方向 | 前缀 | 说明 |
|------|------|------|
| **上行**（STM32 → PC） | `@` | 传感器数据 |
| **上行**（STM32 → PC） | `#OK:` / `#ERR:` | 阈值配置反馈 🆕 |
| **下行**（PC → STM32） | `@` | 控制指令（方向/速度） |
| **下行**（PC → STM32） | `#` | 阈值配置同步 |

---

## 3. 上行协议：传感器数据（STM32 → PC）

### 3.1 帧格式

```
@<温度>,<湿度>,<CO浓度>,<光照强度>,<障碍标志>\r\n
```

### 3.2 字段说明

| 序号 | 字段 | 类型 | 单位 | 示例值 | 说明 |
|------|------|------|------|--------|------|
| 1 | 温度 | float | °C | 25.6 | DHT11/DHT22 等 |
| 2 | 湿度 | float | % | 60.3 | DHT11/DHT22 等 |
| 3 | CO浓度 | float | ppm | 0.5 | MQ-7 等气体传感器 |
| 4 | 光照强度 | float/int | lux | 450 | BH1750 / 光敏电阻 |
| 5 | 障碍标志 | int | - | 0/1 | `1` 表示检测到障碍，`0` 表示无障碍 |

### 3.3 示例

```
@25.6,60.3,0.5,450,0\r\n
@-3.2,85.0,12.7,0,1\r\n
```

### 3.4 发送频率

建议 **500ms ~ 1000ms** 一次，上位机串口线程以 20ms 轮询读取。

## 4. 下行协议一：控制指令（PC → STM32）

### 4.1 帧格式

```
@<指令码>\r\n
```

### 4.2 指令码定义

| 指令码 | 枚举 | 含义 | 说明 |
|--------|------|------|------|
| `0` | CMD_STOP | 停止 | 电机停转 |
| `1` | CMD_FORWARD | 前进 | — |
| `2` | CMD_BACKWARD | 后退 | — |
| `3` | CMD_LEFT | 左转 | — |
| `4` | CMD_RIGHT | 右转 | — |
| `5` | CMD_SPEED_25 | 速度 25% | 对应 SpeedLevel index=0 |
| `6` | CMD_SPEED_50 | 速度 50% | 对应 SpeedLevel index=1 |
| `7` | CMD_SPEED_75 | 速度 75% | 对应 SpeedLevel index=2 |
| `8` | CMD_SPEED_100 | 速度 100% | 对应 SpeedLevel index=3 |
| `9` | CMD_PATROL_START | 启动自动巡逻 | 启动自动巡逻模式 |
| `10` | CMD_PATROL_STOP | 停止自动巡逻 | 停止自动巡逻模式 |

### 4.3 示例

```
@0\r\n   ← 停止
@1\r\n   ← 前进
@5\r\n   ← 速度设为 25%
```

### 4.4 速度指令码公式

```
指令码 = 按钮索引(0~3) + 4
```

## 5. 下行协议二：阈值配置同步（PC → STM32）🆕

### 5.1 帧格式

```
#T<temp_low>,<temp_high>,H<hum_low>,<hum_high>,G<co_warning>,<co_danger>,L<light_low>,<light_high>\r\n
```

### 5.2 字段说明

| 前缀 | 字段 | 类型 | 单位 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| `T` | temp_low | float | °C | -10.0 | 温度安全下限 |
| | temp_high | float | °C | 45.0 | 温度安全上限 |
| `H` | hum_low | float | % | 20.0 | 湿度安全下限 |
| | hum_high | float | % | 90.0 | 湿度安全上限 |
| `G` | co_warning | float | ppm | 30.0 | CO 警告浓度 |
| | co_danger | float | ppm | 50.0 | CO 危险浓度 |
| `L` | light_low | float | lux | 0.0 | 光照安全下限 |
| | light_high | float | lux | 10000.0 | 光照安全上限 |

### 5.3 示例

```
#T-10.0,45.0,H20.0,90.0,G30.0,50.0,L0.0,10000.0\r\n
#T0.0,35.0,H30.0,80.0,G20.0,40.0,L100.0,5000.0\r\n
```

### 5.4 解析规则

1. 判断帧首字符为 `#`
2. 依次查找字母前缀 `T`、`H`、`G`、`L`
3. 每个前缀后跟 **两个逗号分隔的浮点数**（低值在前，高值在后）
4. 解析完成后存入 Flash / EEPROM，并应用到实时告警判断

### 5.5 上位机发送时机

- 用户在告警阈值设置对话框中点击 **"📡 同步到设备"** 按钮
- 仅在串口已连接时发送，未连接时 UI 提示失败

## 5.5 上行协议二：阈值配置反馈（STM32 → PC）🆕

### 5.5.1 帧格式

STM32 收到阈值配置帧后，**必须**返回以下三种反馈之一：

| 反馈帧 | 含义 | 何时返回 |
|--------|------|---------|
| `#OK:Config synced\r\n` | 配置同步成功 | 参数有效且已应用 |
| `#ERR:Invalid params\r\n` | 参数无效 | 低值≥高值等逻辑错误 |
| `#ERR:Parse failed\r\n` | 解析失败 | 帧格式错误、字段缺失 |

### 5.5.2 上位机处理规则

1. 发送 `#T...` 阈值帧后，按钮进入 **"⏳ 同步中…"** 等待状态（禁用）
2. **3 秒超时**：若未收到反馈，提示 **"⚠️ 设备无响应"**
3. 收到 `#OK:` → 按钮变绿 **"✅ 已同步"**，3 秒后恢复
4. 收到 `#ERR:` → 按钮变红 **"❌ 同步失败"**，tooltip 显示具体错误
5. 反馈消息通过消息总线 `serial.config.response` 传递：
   ```json
   {"success": true/false, "detail": "Config synced"}
   ```

### 5.5.3 时序图

```
   上位机                                   STM32
     │                                        │
     │ ── #T-10,45,H20,90,G30,50,L0,10000 ─> │  接收 & 解析
     │    [按钮: ⏳ 同步中…]                    │
     │    [启动 3s 超时定时器]                   │
     │                                        │  验证参数
     │ <──────── #OK:Config synced ────────── │  成功
     │    [按钮: ✅ 已同步]                     │
     │    [3s 后恢复]                           │
     │                                        │
     │ ── #T50,10,H90,20,G60,30,L1000,0 ──>  │  接收 & 解析
     │    [按钮: ⏳ 同步中…]                    │
     │                                        │  发现 low≥high
     │ <──────── #ERR:Invalid params ──────── │  失败
     │    [按钮: ❌ 同步失败]                    │
     │                                        │
```

## 6. 帧区分方法一览

| 收到帧首字符 | 处理方 | 动作 |
|-------------|--------|------|
| `@`（STM32 收到） | STM32 | 解析控制指令 (单数字) |
| `#`（STM32 收到） | STM32 | 解析阈值配置 (多字段) |
| `@`（PC 收到） | 上位机 | 解析传感器数据 (温湿度/CO/光照/障碍标志) |
| `#OK:`（PC 收到） | 上位机 | 阈值配置同步成功 |
| `#ERR:`（PC 收到） | 上位机 | 阈值配置同步失败，显示错误详情 |

---

## 7. 错误处理建议

| 场景 | 建议处理 |
|------|---------|
| 帧不完整（未收到 `\n`） | 继续缓存，等待后续字节 |
| 前缀字母缺失 | 丢弃该帧，打印错误日志 |
| 浮点数解析失败 | 丢弃该帧，使用上次有效值 |
| 字段数量不匹配 | 丢弃该帧（期望 5 字段，兼容旧版 4 字段） |
| 缓冲区溢出 | 清空缓冲区重新同步 |
